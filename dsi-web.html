Decap CMS Guide

Decap CMS provides a user-friendly interface for content management, allowing non-technical users to update website content through a Git-based workflow.

sequenceDiagram participant Editor as Content Editor participant CMS as Decap CMS participant Git as Git Repository participant CI as CI/CD Pipeline participant Site as Live Site Editor->>CMS: Login via OAuth Editor->>CMS: Edit content CMS->>Editor: Show preview Editor->>CMS: Save changes CMS->>Git: Commit to repo Git->>CI: Trigger build CI->>Site: Deploy updates Site->>Editor: Notify of deployment

CMS Configuration

Decap CMS Config (public/admin/config.yml) 
backend:
  name: git-gateway
  branch: main

media_folder: public/media
public_folder: /media

collections:
  - name: pages
    label: Pages
    folder: content/pages
    create: true
    fields:
      - { name: title, label: Title }
      - { name: slug, label: Slug }
      - { name: body, label: Body, widget: markdown }

  - name: products
    label: Products
    folder: content/products
    create: true
    fields:
      - { name: title, label: Title }
      - { name: price, label: Price, widget: number }
      - { name: image, label: Image, widget: image }
      - { name: description, label: Description, widget: markdown }

CMS Best Practices

üìù Content Organization

  • Use clear, descriptive filenames
  • Organize content in logical folders
  • Use consistent frontmatter structure
  • Add alt text for all images

üîÑ Workflow Management

  • Use draft status for work in progress
  • Review content before publishing
  • Use descriptive commit messages
  • Test changes in preview mode

üñºÔ∏è Media Management

  • Optimize images before upload
  • Use consistent naming conventions
  • Organize media in folders
  • Use WebP format when possible

üîí Security

  • Enable two-factor authentication
  • Review user access regularly
  • Use OAuth for authentication
  • Limit admin user count
