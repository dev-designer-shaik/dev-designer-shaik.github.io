Installation Guide

Step-by-step instructions to deploy the AI Ecosystem with all components.

System Requirements

Hardware

  • NVIDIA GPU (RTX 3070 Ti or better)
  • 8GB+ VRAM recommended
  • 16GB+ System RAM
  • 50GB+ Free Disk Space

Software

  • NVIDIA Drivers 535+
  • Docker & Docker Compose
  • NVIDIA Container Toolkit
  • NGC API Key

Initial Configuration

Clone and Configure 
# Clone the repository
cd /home/mdad
git clone <ai-ecosystem-repo>
cd AI-Ecosystem

# Configure NGC API Key
echo "NGC_API_KEY=your_ngc_api_key_here" > nvidia-nim/.env

# Configure Open WebUI
echo "OPENAI_API_BASE_URL=http://host.docker.internal:8000/v1" >> open-webui/.env
echo "OPENAI_API_KEY=nim" >> open-webui/.env

Network Configuration

Docker Network Setup 
# Create shared network
docker network create ai-ecosystem

# Or use existing network from docker-system
docker network connect docker-system_default

Start All Services

Service Startup Script 
# Using the startup script
./scripts/start-all.sh

# Or manually start services
cd nvidia-nim
docker-compose up -d

cd ../open-webui
docker-compose up -d

# Start MCP connectors
cd ../mcp-connectors/qnap
npm start &

Verify Installation

Health Check Commands 
# Check NIM models endpoint
curl -s http://localhost:8000/v1/models

# Verify GPU utilization
nvidia-smi

# Run health check script
./scripts/health-check.sh

# Test Open WebUI
curl -s http://localhost:3000/api/config

Configuration Guide

Detailed configuration options for optimal performance and integration.

NVIDIA NIM Settings

  • Model: mistralai/Mistral-7B-Instruct-v0.2
  • GPU Memory: NIM_GPU_MEMORY_UTILIZATION=0.8
  • Max Tokens: NIM_MAX_TOKENS=4096
  • Port: 8000

Open WebUI Settings

  • Default Model: mistral
  • User Registration: OPENUI_SIGNUP=false
  • Default User: OPENUI_DEFAULT_USER=admin
  • Web Search: ENABLE_RAG=true

MCP Connector Settings

  • QNAP Host: QNAP_HOST=192.168.x.x
  • PLM Server: PLM_SERVER=localhost:3001
  • Frappe MCP: FRAPPE_MCP=localhost:3002
  • iDempiere MCP: IDEMPIERE_MCP=localhost:3003

Performance Tuning

GPU Optimization 
# Adjust GPU memory allocation
export NIM_GPU_MEMORY_UTILIZATION=0.9

# Enable model caching
export NIM_CACHE_SIZE=10GB

# Set concurrent requests
export NIM_MAX_CONCURRENT_REQUESTS=4

Monitoring Script Example

GPU and Service Monitor 
#!/bin/bash
# Continuous monitoring script
while true; do
    clear
    echo "=== GPU Status ==="
    nvidia-smi --query-gpu=utilization.gpu,memory.used,memory.total --format=csv,noheader,nounits

    echo -e "\n=== Service Status ==="
    docker ps --filter "name=open-webui" --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
    docker ps --filter "name=nim" --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

    echo -e "\n=== Recent Logs ==="
    docker logs --tail 5 open-webui 2>&1 | grep -E "(ERROR|WARN)"

    sleep 5
done

Troubleshooting

Common issues and their solutions.

Common Issues

GPU Not Detected

NVIDIA NIM cannot access the GPU

docker: Error response from daemon: could not select device driver
  • Install NVIDIA Container Toolkit
  • Restart Docker daemon
  • Verify GPU drivers: nvidia-smi
  • Check Docker runtime: docker info | grep nvidia

Out of Memory

GPU memory insufficient for model

CUDA out of memory
  • Reduce NIM_GPU_MEMORY_UTILIZATION
  • Use smaller model
  • Close other GPU applications
  • Increase system RAM for swapping

MCP Connector Timeout

AI cannot connect to business systems

Connection timeout after 30s
  • Check network connectivity
  • Verify authentication credentials
  • Check firewall settings
  • Review connector logs

  version: '1.0.0'

        properties: {
          query: { type: 'string' }
        }
      }
    }
  ]
}));

  if (request.params.name === 'get_data') {
    // Implement your logic here
