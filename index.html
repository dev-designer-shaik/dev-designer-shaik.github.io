Getting Started

System Requirements

Hardware

  • RAM: 16GB minimum, 32GB recommended
  • Storage: 100GB SSD free space
  • GPU: RTX 3070 Ti or better (for AI)
  • CPU: 8+ cores recommended

Software

  • Docker 20.10+ & Docker Compose
  • Node.js 18+ & npm
  • Python 3.10+
  • Git 2.30+
  • NVIDIA CUDA Toolkit (GPU users)

Quick Start

Clone and Setup All Repositories 
# Clone all repositories
git clone https://github.com/your-org/Docker-system.git
git clone https://github.com/your-org/AI-Ecosystem.git
git clone https://github.com/your-org/Frappe-MCP.git
git clone https://github.com/your-org/idempiere-mcp.git
git clone https://github.com/your-org/DSI-WEB-N.git
git clone https://github.com/your-org/dev-designer-shaik.github.io.git

# Navigate to Docker-system and start services
cd Docker-system
docker-compose up -d

# Setup AI Ecosystem
cd ../AI-Ecosystem
./setup.sh

# Install MCP servers
cd ../Frappe-MCP
pip install -r requirements.txt
python mcp.py

cd ../idempiere-mcp
pip install -r requirements.txt
python server.py

Environment Configuration

Create a .env file in each repository with the following variables:

Docker-system

  • POSTGRES_PASSWORD
  • RABBITMQ_USER
  • REDIS_PASSWORD
  • OPENAI_API_KEY

AI-Ecosystem

  • NVIDIA_API_KEY
  • MODEL_NAME
  • GPU_ENABLED
  • WEBUI_PORT

Key API Endpoints

Frappe MCP Endpoints

  • POST /api/v1/query/doctypes - Query any doctype
  • POST /api/v1/query/relationships - Get relationships
  • POST /api/v1/files/list - List attachments
  • POST /api/v1/analytics/dashboard - Get dashboard data

iDempiere MCP Endpoints

  • POST /api/v1/execute_query - Execute SQL query
  • POST /api/v1/get_products - Get product data
  • POST /api/v1/get_business_partners - Get B2B data
  • POST /api/v1/list_tables - List all tables

Troubleshooting

Common Issues

Services won't start

Check if ports are available and Docker is running.

sudo netstat -tulpn | grep :3000

GPU not detected

Verify NVIDIA drivers and CUDA installation.

nvidia-smi

Database connection failed

Check PostgreSQL status and credentials.

docker exec -it postgres psql -U postgres

Debug Tools

  • Use docker-compose logs [service] to view logs
  • Check health with curl http://localhost:3000/health
  • Monitor resources with docker stats
  • Test API endpoints with Postman or curl
