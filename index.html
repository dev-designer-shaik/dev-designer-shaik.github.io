3.7 Workflow Steps

Step 1: Intelligent File Detection & Metadata Extraction

  • Trigger: Google Drive API detects new file in Palace directories
  • Path Analysis: Extract product name, SKU, and category from folder structure
  • File Validation: Verify file type, size, and naming conventions
  • MCP Assignment: Generate Model Context Protocol ID for tracking
  • Initial Logging: Create processing record in PostgreSQL

Step 2: AI-Powered Content Analysis

  • Weaviate Processing: Generate semantic embeddings and visual analysis
  • OpenAI Tagging: Extract colors, composition, style, and brand compliance
  • Quality Assessment: Rate image quality and brand alignment
  • Metadata Generation: Create comprehensive metadata package
  • Vector Storage: Store embeddings for semantic search

Step 3: Multi-Platform Distribution

  • Cloudinary Optimization: Generate multiple formats and sizes
  • Naming Convention Application: Apply standardized file naming
  • Search Index Updates: Update all search databases
  • Google Drive Enhancement: Add rich descriptions and properties
  • System Integration: Update ERPNext, WooCommerce, and Webflow

function searchAssets(searchTerm) {
  // Advanced search function with multiple criteria
  const indexFile = DriveApp.getFilesByName('searchable_media_index.json').next();
  const indexData = JSON.parse(indexFile.getBlob().getDataAsString());
  
  let results = [];
  
  for (const [productKey, productData] of Object.entries(indexData.products)) {
    productData.assets.forEach(asset => {
      const searchableContent = [
        productData.product_info.name,
        productData.product_info.sku,
        productData.product_info.category,
        ...asset.ai_metadata.tags,
        asset.ai_metadata.description
      ].join(' ').toLowerCase();
      
      if (searchableContent.includes(searchTerm.toLowerCase())) {
        results.push({
          product: productData.product_info.name,
          sku: productData.product_info.sku,
          file: asset.file_info.processed_name,
          quality: asset.ai_metadata.quality_score,
          drive_link: `https://drive.google.com/file/d/${asset.file_info.drive_file_id}`,
          cloudinary_url: asset.cloudinary_urls.web_optimized
        });
      }
    });
  }
  
  return results.sort((a, b) => b.quality - a.quality);
}

Google Cloud Function Deployment Guide

This section details the deployment of a Google Cloud Function to act as a webhook for Google Drive changes, forwarding them to Google Pub/Sub. This function is crucial for triggering real-time media ingestion workflows.

1. `requirements.txt`


google-cloud-pubsub==2.15.0
google-cloud-storage==2.10.0

2. `main.py` (Python Code for `drive_webhook_to_pubsub`)


import os
import base64
from google.cloud import pubsub_v1

PROJECT_ID = os.environ.get("GCP_PROJECT_ID")
TOPIC_ID = os.environ.get("GCP_PUBSUB_TOPIC_ID")

def drive_webhook_to_pubsub(request):
    publisher = pubsub_v1.PublisherClient()
    topic_path = publisher.topic_path(PROJECT_ID, TOPIC_ID)

    # Extract relevant headers from the Drive webhook notification
    headers = dict(request.headers)
    message_data = {
        "X-Goog-Channel-ID": headers.get("X-Goog-Channel-ID"),
        "X-Goog-Channel-Token": headers.get("X-Goog-Channel-Token"),
        "X-Goog-Channel-Expiration": headers.get("X-Goog-Channel-Expiration"),
        "X-Goog-Message-Number": headers.get("X-Goog-Message-Number"),
        "X-Goog-Resource-State": headers.get("X-Goog-Resource-State"),
        "X-Goog-Resource-ID": headers.get("X-Goog-Resource-ID"),
        "X-Goog-Resource-URI": headers.get("X-Goog-Resource-URI"),
        "Content-Type": headers.get("Content-Type"),
        "Content-Length": headers.get("Content-Length"),
        "Body": request.get_data(as_text=True) # Capture the body if any
    }

    # Convert message_data to a string and then to bytes
    data_str = str(message_data)
    data_bytes = data_str.encode("utf-8")

    # Publish the message to Pub/Sub
    future = publisher.publish(topic_path, data_bytes)
    print(f"Published message ID: {future.result()}")

    return "OK", 200

3. Deployment Commands


# Deploy the Cloud Function
gcloud functions deploy drive-webhook-to-pubsub \
    --runtime python39 \
    --trigger-http \
    --allow-unauthenticated \
    --entry-point drive_webhook_to_pubsub \
    --set-env-vars GCP_PROJECT_ID=your-gcp-project-id,GCP_PUBSUB_TOPIC_ID=your-pubsub-topic-id

# After deployment, get the function's trigger URL
gcloud functions describe drive-webhook-to-pubsub --format="value(httpsTrigger.url)"
